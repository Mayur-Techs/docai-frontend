FAQ

Common questions

What types of PDFs does it work with? +
Any digital PDF — GST invoices, freight bills, purchase orders, customs documents, commercial invoices. Scanned PDFs (image-only) have lower accuracy because there is no text layer to parse. For scanned documents, AI extraction handles it but confidence scores will be lower.
Do you store my invoice data? +
Yes — extracted data is stored in your account so you can search, export, and reprocess it later. The original PDF file is stored for 30 days then deleted. Extracted JSON is kept for the lifetime of your account. You can delete any document at any time via the API or dashboard.
How accurate is the extraction? +
For clean digital PDFs (the majority of invoices sent by email): 90–96% accuracy. For complex or scanned PDFs: 65–80%. Every extraction includes a confidence score — anything below 70% is flagged so you know to verify it. GSTIN extraction is 99%+ accurate because it has a fixed pattern.
What is "rules-first" extraction? +
Most document AI tools send every PDF directly to an LLM (expensive, slow). We parse the PDF with pdfplumber first, then run regex patterns for known fields like GSTIN, invoice numbers, and amounts. If we get 70%+ confidence from code alone, we never call the AI. This makes extraction faster, cheaper, and more reliable for standard Indian invoice formats.
Can I integrate this with Tally or my ERP? +
Business and Enterprise plans include a REST API that returns structured JSON. You can use n8n (included) to push that JSON into Tally, SAP, or any ERP with a webhook or API endpoint. The n8n workflow template is included and takes about 30 minutes to set up with your ERP.
Is there a free trial for paid plans? +
The Business plan includes a 7-day free trial with no credit card required. If you process your first 50 invoices and it doesn't work for your document types, you pay nothing. Just email us and we'll close the account.
